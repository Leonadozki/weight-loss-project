# AI 陪跑流程设计文档

## 1. 设计目标

减脂方案的核心痛点不是"不知道怎么做"，而是"做不到"。AI 陪跑的设计目标是：

1. **降低执行门槛**：把大目标拆解为每日可执行的小任务
2. **提供即时反馈**：用户打卡后立刻获得 AI 回应，形成正反馈循环
3. **动态调整方案**：根据实际执行数据，避免计划与实际脱节
4. **情感陪伴**：在用户想放弃时给予鼓励，在松懈时适度提醒

---

## 2. 用户旅程

```
Day 0: 完成评估 → 生成方案 → 进入陪跑
    ↓
Day 1-N: 每日打卡（5分钟）→ AI 即时反馈 → 查看进度
    ↓
Day 7: 周度回顾 → 方案微调 → 设定下周目标
    ↓
Day N: 达成目标 → 里程碑庆祝 → 进入维持期
```

---

## 3. 每日打卡流程

### 3.1 打卡维度（5维模型）

| 维度 | 输入方式 | 数据来源 |
|------|----------|----------|
| 体重 | 数字输入 | 用户称重 |
| 饮食 | 0-5 分自评 + 简要记录 | 用户主观评估 |
| 运动 | 是否完成 + 时长 | 用户记录 |
| 睡眠 | 小时数 | 用户估计 |
| 心情 | 5 级表情选择 | 用户主观 |

### 3.2 打卡流程图

```
打开打卡页
    ↓
显示今日待打卡项（已完成的打勾）
    ↓
用户逐项填写
    ↓
点击"完成打卡"
    ↓
系统生成 AI 日报（本地规则引擎，模拟 AI）
    ↓
展示日报卡片（总结 + 鼓励 + 建议）
    ↓
用户可继续对话（追问 / 倾诉）
```

## 4. AI 日报生成规则（MVP 版）

MVP 阶段使用本地规则引擎模拟 AI，规则如下：

### 4.1 体重分析

| 场景 | 反馈模板 |
|------|----------|
| 比昨日下降 | "体重下降 X kg，继续保持！" |
| 比昨日持平 | "体重持平，这是平台期的正常现象。" |
| 比昨日上升 <0.5 | "小幅波动不用焦虑，关注长期趋势。" |
| 比昨日上升 >=0.5 | "⚠️ 体重有明显回升，回顾下昨日的饮食和运动情况？" |
| 连续3天下降 | "太棒了！连续3天下降，你的身体正在适应新节奏。" |
| 连续3天上升 | "⚠️ 连续3天上升，建议检查一下最近的饮食结构。" |

### 4.2 饮食分析

| 得分 | 反馈 |
|------|------|
| 5 | "饮食执行完美，给你点赞！" |
| 4 | "饮食控制得不错，偶尔的小奖励也是可以的。" |
| 3 | "饮食中规中矩，尝试下一顿再控制一下份量？" |
| 2 | "今天饮食有些放松，明天我们一起加油。" |
| 0-1 | "⚠️ 今天的饮食偏离计划较多，发生了什么吗？" |

### 4.3 运动分析

| 场景 | 反馈 |
|------|------|
| 完成且超量 | "超额完成运动目标，太强了！注意别过度疲劳。" |
| 刚好完成 | "按计划完成运动，执行力满分！" |
| 部分完成 | "完成了部分运动，比不做好多了。" |
| 未完成 | "今天没运动？没关系，明天补上就好。" |

### 4.4 综合鼓励语池

根据心情选择不同的鼓励语风格：
- great/good: 积极强化型
- neutral: 温和陪伴型
- tired/bad: 理解安慰型

## 5. 进度看板设计

### 5.1 展示内容

1. **体重曲线**：近 30 天体重变化折线图
2. **打卡日历**：当月日历，每天显示打卡状态（完整/部分/未打卡）
3. **关键指标**：
   - 已坚持天数
   - 累计减重
   - 目标达成率 %
   - 平均饮食得分
   - 运动完成率

### 5.2 里程碑系统

| 触发条件 | 奖励 |
|----------|------|
| 连续打卡 7 天 | "周达人" 徽章 |
| 连续打卡 30 天 | "月冠军" 徽章 |
| 减重达 5% | "初战告捷" 徽章 |
| 减重达 10% | "蜕变之星" 徽章 |
| 达成目标体重 | "减脂大师" 徽章 + 庆祝动画 |

## 6. 周度回顾流程

每周一自动触发（或用户手动查看）：

1. **数据汇总**：上周平均体重、总运动时长、平均饮食分
2. **趋势判断**：体重趋势（下降/持平/上升）
3. **方案调整建议**：
   - 如果进展顺利 → 保持当前计划
   - 如果平台期 → 建议调整运动类型或热量
   - 如果反弹 → 分析原因 + 调整建议
4. **下周目标设定**：基于当前进度动态调整

## 7. 页面交互设计

### 7.1 /coach — 陪跑主页

- 顶部：今日打卡状态速览（5个维度的小卡片，已完成绿色，未完成灰色）
- 中部：AI 教练欢迎语 + 今日一句话建议
- 底部：快捷入口（去打卡 / 看进度 / 周回顾）

### 7.2 /coach/checkin — 打卡页

- 步骤式表单，每步一个维度
- 完成后展示 AI 日报卡片（带动画）
- 提供"和教练聊聊"按钮进入对话

### 7.3 /coach/progress — 进度页

- 体重曲线图（用 div 模拟简单折线）
- 打卡日历
- 里程碑展示墙
- 关键指标卡片

## 8. 后续升级路线

| 阶段 | 升级内容 |
|------|----------|

| v0.6 | 接入运动数据（Apple Health / 小米运动） |
